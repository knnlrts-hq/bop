1.5 Union, Intersection, Difference¶

Just as numbers are combined with operations such as addition, subtraction and multiplication, there are various operations that can be applied to sets. The Cartesian product (defined in Section 1.2) is one such operation; given sets \(A\) and \(B\), we can combine them with \(\times\) to get a new set \(A \times B\). Here are three new operations called union, intersection and difference.

Definition 1.5: Suppose A and \(B\) are sets. The union of \(A\) and \(B\) is the set \(A \cup B = \left\{ x : x \in A \right.\) or \(x \in B \}\). The intersection of \(A\) and \(B\) is the set \(A \cap B = \left\{ x : x \in A \right.\) and \({ x \in B } \}\). The difference of \(A\) and \(B\) is the set \(A - B = \left\{ x : x \in A \right.\) and \({ x \notin B } \}\).

In words, the union \(A \cup B\) is the set of all things that are in \(A\) or in \(B\) (or in both). The intersection \(A \cap B\) is the set of all things in both \(A\) and \(B\). The difference \(A - B\) is the set of all things that are in \(A\) but not in \(B\).

Example 1.8 Suppose \(A = \{ a , b , c , d , e \}\) , \({ B } = \left\{ d , e , f \right\}\) and \(C = \{ 1 , 2 , 3 \}\)

  1. \(A \cup B = \{ a , b , c , d , e , f \}\)

  2. \(A \cap B = \left\{ d , e \right\}\)

  3. \(A-B = \{a,b,c\}\)

  4. \(B-A = \{f\}\)

  5. \((A-B) \cup (B-A) = \{a,b,c,f\}\)

  6. \(A \cup C = \{ a , b , c , d , e , 1,2,3 \}\)

  7. \(A \cap C = \varnothing\)

  8. \(A - C = \{ a , b , c , d , e \}\)

  9. \(( A \cap C ) \cup ( A - C ) = \left\{ a , b , c , d , e \right\}\)

  10. \(( A \cap B ) \times B = \big\{ (d,d),(d,e),(d,f),(e,d),(e,e),(e,f) \big\}\)

  11. \(( A \times C ) \cap ( B \times C ) = \big\{ (d,1),(d,2),(d,3),(e,1),(e,2),(e,3) \big\}\)

Parts 12–15 use interval notation (Section 1.1), so \([ 2 , 5 ] = \left\{ x \in \mathbb { R } : 2 \leq x \leq 5 \right\}\), etc. Sketching these on the number line may aid your understanding. 12. \([ 2 , 5 ] \cup [ 3 , 6 ] = [ 2 , 6 ]\) 13. \([2, 5] \cap [3, 6] = [3, 5]\) 14. \([2, 5] − [3, 6] = [2, 3)\) 15. \([ 0 , 3 ] - [ 1 , 2 ] = [ 0 , 1 ) \cup ( 2 , 3 ]\)

Observe that for any sets \(X\) and \(Y\) it is always true that \(X \cup Y = Y \cup X\) and \(X \cap Y = Y \cap X\), but in general \(X - Y \neq Y - X\).

Example 1.9: Let \(A = \left\{ ( x , x ^ { 2 } ) : x \in \mathbb { R } \right\}\) be the graph of the equation \(y = x ^ { 2 }\) and let \(B = \left\{ ( x , x + 2 ) : x \in \mathbb { R } \right\}\) be the graph of the equation \(y = x + 2\). These sets are subsets of \(\mathbb { R } ^ { 2 }\). They are sketched together in Figure 1.5(a). Figure 1.5(b) shows \(A \cup B\), the set of all points \(( x , y )\) that are on one (or both) of the two graphs. Observe that \(A \cap B = \left\{ ( - 1 , 1 ) , ( 2 , 4 ) \right\}\) consists of just two elements, the two points where the graphs intersect, as illustrated in Figure 1.5(c). Figure 1.5(d) shows \(A - B\), which is the set \(A\) with “holes” where \(B\) crossed it. In set builder notation, we could write \(A \cup B = \left\{ ( x , y ) : x \in \mathbb { R } , y = x ^ { 2 } \right.\) or \(y = x + 2 \}\) and \(A - B = \left\{ ( x , x ^ { 2 } ) : x \in \mathbb { R } - \left\{ - 1 , 2 \right\} \right\}\) (\(\mathbb{R}\) without \(-1\) and \(2\)).

def semicircle_points(side, n=181):
    """
    Return points on the unit semicircle.
    side='right': from (0,-1) to (0,1) through (1,0)
    side='left' : from (0,-1) to (0,1) through (-1,0)
    """
    if side == "right":
        theta = np.deg2rad(np.linspace(-90, 90, n))
    elif side == "left":
        theta = np.deg2rad(np.linspace(270, 90, n))
    else:
        raise ValueError("side must be 'right' or 'left'")
    return np.c_[np.cos(theta), np.sin(theta)]

Sketch the sets \(X = \left\{ ( x , y ) \in \mathbb { R } ^ { 2 } : x ^ { 2 } + y ^ { 2 } \leq 1 \right\}\) and \(Y = \left\{ ( x , y ) \in \mathbb { R } ^ { 2 } : - 1 \leq y \leq 0 \right\}\) on \(\mathbb { R } ^ { 2 }\). Answer

\(X\cup Y = \{(x,y):x^2+y^2\le1\}\ \cup\ \{(x,y):-1\le y\le0\}\) \(X\cap Y = \{(x,y):x^2+y^2\le1,\ -1\le y\le0\}\) \(X-Y = \{(x,y):x^2+y^2\le1,\ y>0\}\) \(Y-X = \{(x,y):-1\le y\le0,\ x^2+y^2>1\}\)

def style_axes(ax, title):
    ax.axhline(0, color="black", linewidth=0.5)
    ax.axvline(0, color="black", linewidth=0.5)

    ax.grid(True, linestyle=":")
    ax.set_title(title, fontsize=11)
    ax.set_axisbelow(True)

def rgba(c, alpha):
    return (c[0], c[1], c[2], alpha)

def semicircle_points(side, n=181):
    """
    Return points on the unit semicircle.

    side='right': from (0,-1) to (0,1) through (1,0)
    side='left' : from (0,-1) to (0,1) through (-1,0)
    side='up'   : from (-1,0) to (1,0) through (0,1)
    side='down' : from (-1,0) to (1,0) through (0,-1)
    """
    if side == "right":
        theta = np.deg2rad(np.linspace(-90, 90, n))
    elif side == "left":
        theta = np.deg2rad(np.linspace(270, 90, n))
    elif side == "up":
        theta = np.deg2rad(np.linspace(180, 0, n))
    elif side == "down":
        theta = np.deg2rad(np.linspace(180, 360, n))
    else:
        raise ValueError("side must be one of 'right', 'left', 'up', 'down'")
    return np.c_[np.cos(theta), np.sin(theta)]

def curved_region_patch(vertices, facecolor, edgecolor="none", linewidth=1.0):
    verts = np.asarray(vertices, dtype=float)
    verts_closed = np.vstack([verts, verts[0]])
    codes = [Path.MOVETO] + [Path.LINETO] * (len(verts) - 1) + [Path.CLOSEPOLY]
    path = Path(verts_closed, codes)
    return PathPatch(path, facecolor=facecolor, edgecolor=edgecolor, linewidth=linewidth)

def region_with_hole_patch(outer_vertices, inner_vertices, facecolor, edgecolor="none", linewidth=1.0):
    """
    Create a filled region with one hole.
    The inner path should be given with opposite orientation to the outer path.
    """
    outer = np.asarray(outer_vertices, dtype=float)
    inner = np.asarray(inner_vertices, dtype=float)
    outer_closed = np.vstack([outer, outer[0]])
    inner_closed = np.vstack([inner, inner[0]])
    verts = np.vstack([outer_closed, inner_closed])
    codes = (
        [Path.MOVETO] + [Path.LINETO] * (len(outer) - 1) + [Path.CLOSEPOLY] +
        [Path.MOVETO] + [Path.LINETO] * (len(inner) - 1) + [Path.CLOSEPOLY]
    )
    path = Path(verts, codes)
    return PathPatch(path, facecolor=facecolor, edgecolor=edgecolor, linewidth=linewidth)

  1. Is the statement \(( \mathbb { R } \times \mathbb { Z } ) \cap ( \mathbb { Z } \times \mathbb { R } ) = \mathbb { Z } \times \mathbb { Z }\) true or false? What about the statement \(( \mathbb { R } \times \mathbb { Z } ) \cup ( \mathbb { Z } \times \mathbb { R } ) = \mathbb { R } \times \mathbb { R }\)?

Answer

The first statement is true; an ordered pair lies in both sets if and only if both coordinates are integers. The second statement is false; for example \((0.5,0.5) \in \mathbb{R} \times \mathbb{R}\) but \((0.5,0.5) \notin (\mathbb{R} \times \mathbb{Z}) \cup (\mathbb{Z} \times \mathbb{R})\). \(( \mathbb { R } \times \mathbb { Z } ) \cup ( \mathbb { Z } \times \mathbb { R } )\) creates a checkerboard pattern, not the full \(\mathbb{R}^{2}\) plane.

  1. Do you think the statement \(( \mathbb { R } - \mathbb { Z } ) \times \mathbb { N } = ( \mathbb { R } \times \mathbb { N } ) - ( \mathbb { Z } \times \mathbb { N } )\) is true, or false? Justify.

Answer

True. In general, for sets \(A,B,C\) it holds that: \((A-B)\times C=(A\times C)-(B\times C)\). \((x,y)\in (A-B)\times C\) \(\iff (x\in A\ \text{and}\ x\notin B)\ \text{and}\ y\in C\) \(\iff (x,y)\in A\times C\ \text{and}\ (x,y)\notin B\times C\) Taking \(A=\mathbb{R}\), \(B=\mathbb{Z}\), and \(C=\mathbb{N}\) yields the claimed identity.
